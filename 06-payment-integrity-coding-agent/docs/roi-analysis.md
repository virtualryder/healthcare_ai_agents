# Agent 06 — ROI Analysis: Payment Integrity & Coding

> Decision-support accelerator. SA-fillable model; external stats are source-class-tagged in
> `../../gtm/HPP-DECK-SOURCES.md`.

## Two-sided value
For **providers**, accurate coding pre-bill reduces the denials Agent 01 reworks (coding is a
top denial root cause) and lowers False Claims Act exposure. For **payers**, pre- and post-pay
integrity review surfaces overpayments, duplicates, and unsupported-necessity claims —
measurable recoveries.

| Driver | Note | Source class |
|---|---|---|
| Coding-driven denials | A leading initial-denial root cause (CO-16, CO-11, etc.) | industry-research |
| FCA / OIG exposure | Upcoding and unsupported necessity are enforcement priorities | gov |
| Payment-integrity recoveries | Overpayment/duplicate detection is a measurable payer outcome | sector-press |

## Value model (fill per customer)
```
Claims reviewed per month                 C
Coder/analyst minutes per review          m   (e.g. 12)
Agent deflection of triage/edit work      e   (e.g. 0.5)
Flag rate (issues found)                  f   (e.g. 0.06)
Avg recovery / avoided-denial value       R
Loaded analyst cost per hour              c

Analyst hours returned/month ≈ C * m/60 * e
Issues surfaced for review/month ≈ C * f
```
Value shows up as recovered analyst capacity, fewer coding-driven denials (compounding with
Agent 01), and surfaced recoveries — all confirmed by a human before any action.

## What is NOT claimed
The agent does not recoup, adjust payment, submit, or assert fraud. A flag is a prompt for a
human reviewer. Value is measured in a pilot, not assumed.
